# 智能检索工作台重构实施计划

日期：2026-07-13  
分支：`codex/search-workbench-redesign`

## 1. 目标

将现有智能检索页面重构为单一、完整的知识检索工作台，覆盖：

- 初始发现态：搜索、最近搜索、热门搜索、真实知识库入口、最近更新。
- 结果工作态：检索策略、筛选、排序、分页、结果聚合与权限提示。
- 阅读决策态：预览、打开原文、下载；仅在问答上下文契约接通后展示“向 AI 提问”。
- 完整状态：加载、空数据、无结果、错误、无下载权限和移动端适配。

## 2. 设计约束

- 保留全局一级菜单“智能检索”，移除检索模块内部固定二级侧栏。
- 页面只有一个主操作：搜索。
- 首页推荐内容必须来自真实接口，不使用硬编码分类 ID 或虚构统计。
- 未接通的高级字段不展示只读占位控件。
- 普通用户默认只看到有权访问的内容；权限范围不是主筛选项。
- URL 保存关键词、检索策略、筛选、排序、页码和视图状态。
- “预览”“打开原文”“下载”“向 AI 提问”必须具有不同语义和行为。
- 交互控件具备键盘焦点、可访问名称、加载与禁用状态。

## 3. 页面结构

### 3.1 初始发现态

1. 页面标题与简短说明。
2. 主搜索框：知识库范围、关键词、搜索按钮、检索策略入口。
3. 常用知识库：由文件夹/知识库接口生成。
4. 最近搜索：关键词、搜索时间、结果数、删除与清空。
5. 热门搜索：时间范围、搜索次数；无数据时降级为最近更新。
6. 最近更新：真实文档数据。

### 3.2 结果工作态

1. 吸顶搜索栏。
2. 已选条件标签及一键清空。
3. 横向紧凑筛选栏；扩展说明按需展开，小屏自动换行。
4. 结果摘要、排序和列表/宫格切换。
5. 文档级结果，展示多个命中片段而不是重复文档。
6. 真实分页、上一页/下一页和页码状态。
7. 预览/打开/下载操作；暂不展示尚未接通文档上下文的 AI 问答入口。

### 3.3 状态设计

- 首次加载：内容骨架屏。
- 搜索中：保持既有结果并显示局部进度。
- 无结果：关键词建议、放宽条件、清空筛选。
- 接口失败：错误原因、重试按钮。
- 热门/历史为空：解释数据产生方式并提供下一步。
- 无下载权限：允许查看时保留预览，明确隐藏或禁用下载。

## 4. 多 Agent 任务拆分

### A. 高风险：检索契约与后端闭环（5.6 SOL）

写入范围：

- `packages/schemas/src/search.ts`
- `apps/api/src/modules/search/search.controller.ts`
- `apps/api/src/modules/search/search.service.ts`
- `apps/api/src/modules/search/search.service.spec.ts`

任务：

1. 为列表检索补充可选 `mode`，保持 `hybrid` 默认值。
2. 修正关键词检索固定前 10 条被当作总数的问题。
3. 返回明确分页字段：`page`、`pageSize`、`total`、`hasMore`。
4. 保持租户、权限、归档和 `searchable` 过滤不被绕过。
5. 保证筛选检索和关键词检索都遵守相同分页语义。
6. 保持历史记录和搜索事件每次显式搜索只记录一次。
7. 补充模式、分页、权限过滤和空查询测试。
8. 搜索交互事件增加频率限制、目标文档权限校验和短期签名结果令牌，事件必须绑定真实返回的用户、租户、关键词与文档，并在短窗口内去重。
9. 无效或无权知识库范围不得退化为全租户搜索。
10. 候选集达到检索上限时返回显式截断标记，页面不冒充全量精确总数。

验收：

- 模式选择实际传入检索算法。
- 第二页可以取得不同结果。
- `total` 不再恒等于当前页条数。
- 所有新增/既有后端测试通过。

### B. 中风险：前端页面状态编排（5.6 Terra）

写入范围：

- `apps/web/src/app/(dashboard)/search/SearchPageClient.tsx`
- `apps/web/src/app/(dashboard)/search/page.spec.ts`
- `apps/web/src/types/api/search.ts`
- `apps/web/src/lib/api/endpoints/search.ts`
- `apps/web/src/services/search.ts`

任务：

1. 建立 landing/results 两种主状态，删除固定二级侧栏依赖。
2. URL 同步关键词、mode、筛选、排序、页码和视图。
3. 分类改用真实文件夹/知识库数据，不再使用硬编码英文 ID。
4. 首页加载历史、热门、知识库和最近文档，失败时局部降级。
5. 接入分页并在查询变化时回到第 1 页。
6. 记录结果点击、查看和下载事件。
7. 区分预览、打开原文和下载处理函数。
8. 保留请求序号保护，避免旧请求覆盖新结果。
9. 为 AI 问答跳转预留带文档和关键词的 URL 参数。

验收：

- 刷新页面后搜索状态不丢失。
- 进入结果态后不存在失效的历史/热门菜单。
- 分类选择不会同时强制添加同名关键词。
- 页码、排序、模式变化触发一次对应请求。

### C. 低风险：展示组件与可访问性（5.6 Luna）

写入范围：

- `apps/web/src/components/search/SearchLanding.tsx`
- `apps/web/src/components/search/SearchFilters.tsx`
- `apps/web/src/components/search/SearchResultsToolbar.tsx`
- `apps/web/src/components/search/SearchResultList.tsx`
- `apps/web/src/components/search/SearchResultGrid.tsx`
- `apps/web/src/components/search/HotSearchPanel.tsx`
- `apps/web/src/components/search/SearchHistoryPanel.tsx`
- 新增的 `apps/web/src/components/search/*` 纯展示组件与组件测试

任务：

1. 首页改成搜索主区、知识库、历史、热门和最近更新的响应式网格。
2. 筛选组件改为真实可用字段，删除只读高级条件。
3. 结果项强化标题、路径、高亮片段、更新时间、类型和权限层级。
4. 增加加载骨架、无结果、局部失败和空数据展示组件。
5. 增加分页组件和已选条件标签。
6. 所有图标按钮提供文本或 `aria-label`，焦点样式可见。
7. 375/768/1024/1440 宽度下不产生页面级横向滚动。

验收：

- 不依赖 API 的纯组件可单独渲染。
- 不显示未实现功能。
- 空白区域由真实、有行动价值的内容填充。

### D. 高风险：文件预览与下载语义（5.6 SOL）

写入范围：

- `apps/api/src/modules/qa/qa.controller.ts`
- 对应 QA controller 测试
- `apps/web/src/lib/api/endpoints/qa.ts`
- `apps/web/src/services/qa.ts`
- 对应 QA endpoint 测试

任务：

1. 将文件权限语义拆清：内联查看校验 `VIEW`，附件下载校验 `DOWNLOAD`。
2. 为文件接口增加明确的 `inline` 与 `attachment` 响应语义。
3. 前端通过统一鉴权客户端获取 Blob，兼容 Bearer 与 Cookie 会话。
4. 验证中文文件名编码、缓存头和权限路径不被绕过。

验收：

- “打开原文”在浏览器内预览。
- “下载”触发附件下载。
- 两条路径共享同一租户与文档访问控制入口，并按动作分别校验查看/下载能力。

## 5. 主 Agent 集成任务

1. 冻结跨 Agent 类型契约，处理并行改动接口差异。
2. 集成 PDF/Markdown 预览或文档详情面板。
3. 检查下载接口是否使用附件语义，必要时降级为“打开原文”。
4. 检查 AI 问答是否能接收文档上下文；未接通时隐藏入口而非假跳转。
5. 去除废弃 `SearchSectionNav` 的引用；确认无用后再删除文件。
6. 保留当前工作树中与搜索无关的既有用户改动。

## 6. 验证矩阵

### 自动化

- Search schema/service/controller 单元测试。
- Web search endpoint 测试。
- Search page 结构与状态测试。
- TypeScript `--noEmit`。
- Web production build。

### 手工交互

- 空账号、无历史、无热门数据。
- 关键词检索、仅筛选检索、无结果检索。
- mode、排序、筛选、翻页和浏览器返回。
- PDF、Markdown 和不可预览文件。
- 可查看不可下载文档。
- 375、768、1024、1440 宽度。
- 键盘 Tab、Enter、Escape 和可见焦点。

## 7. 完成标准

- 页面无固定二级检索侧栏。
- 无硬编码分类 ID 和不可用高级条件。
- 搜索结果支持真实分页与模式选择。
- 首页至少包含三类真实数据内容，且每类有独立空/错状态。
- 查看、下载、AI 问答入口不存在伪功能。
- 类型检查、相关测试和生产构建通过。
