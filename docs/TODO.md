## 贡献指南

### 认领任务

1. 在 GitHub Issues 中创建或选择任务
2. 创建分支：`feature/xxx` 或 `fix/xxx`
3. 完成任务后提交 PR
4. 至少 1 人 review 后合并

### 代码规范

- 遵循项目现有的代码风格
- 新功能必须包含单元测试
- 更新 API 需要同步更新文档

### Commit Message 规范

```
<type>(<scope>): <subject>

feat(search): add hybrid search mode
fix(auth): resolve token refresh issue
docs(readme): update deployment guide
refactor(api): optimize database queries
```
