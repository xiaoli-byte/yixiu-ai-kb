## 1. 系统概述

### 1.1 系统功能

本系统是一个**企业级 AI 知识库**，提供四大核心功能：

| 功能模块 | 说明 |
|---------|------|
| **文档管理** | 上传、解析、切片、向量化和存储企业文档 |
| **智能检索** | 混合检索：关键词 + 语义向量 + RRF 融合 |
| **AI 问答** | 基于文档的检索增强问答 (RAG)，支持流式响应 |
| **知识图谱** | 可视化展示文档、实体和关系网络 |

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────┐
│                    前端 (Next.js)                    │
│  文档管理 │ 智能检索 │ AI 问答 │ 知识图谱 │ 系统设置   │
└─────────────────────┬───────────────────────────────┘
                      │ HTTP / SSE
┌─────────────────────▼───────────────────────────────┐
│                   后端 (NestJS)                      │
│  Auth │ Documents │ Search │ QA │ Graph │ Queue    │
└───────┬────────────┬──────────┬─────────┬──────────┘
        │            │          │         │
   ┌────▼────┐  ┌───▼────┐ ┌───▼────┐ ┌─▼──────┐
   │MinIO    │  │Postgres│ │ Redis  │ │ Neo4j  │
   │(文件存储)│  │+pgvector│ │(队列) │ │(图谱)  │
   └─────────┘  └────────┘ └────────┘ └────────┘
        │
   ┌────▼──────┐
   │PaddleOCR  │ 图片 / 扫描 PDF OCR
   └───────────┘
```

## 2. 登录与认证

### 2.1 登录系统

1. 访问系统地址 (默认 `http://localhost:8888`)
2. 输入邮箱和密码
3. 点击「登录」按钮

**演示账号：**
```
邮箱：admin@demo.com
密码：demo123
```

### 2.2 Token 机制

系统使用 **JWT 双 Token 机制**：

| Token | 有效期 | 用途 |
|-------|--------|------|
| Access Token | 7 天 | API 访问鉴权 |
| Refresh Token | 30 天 | 刷新 Access Token |

登录成功后，Token 会自动存储在前端，API 请求会自动携带。

### 2.3 退出登录

点击右上角用户头像 → 「退出登录」，清除本地 Token。

## 3. 文档管理

### 3.1 页面入口

点击左侧导航栏「文档管理」进入文档列表页面。

### 3.2 上传文档

**支持的文件格式：**
- PDF (.pdf)
- Markdown / 文本 (.md, .markdown, .txt, .csv, .json, .jsonl)
- Word (.docx, .doc, .docm)
- Excel (.xlsx, .xls, .xlsm)
- PowerPoint (.pptx, .ppt, .pptm)
- 图片 OCR (.png, .jpg, .jpeg, .jpe, .jfif, .webp, .bmp, .tif, .tiff)
- 音频/音视频 (.wav, .mp3, .m4a, .aac, .flac, .ogg, .opus, .webm, .amr, .wma, .mp4, .mov, .mkv)

**上传步骤：**

1. 点击右上角「上传文档」按钮
2. 在弹窗中选择：
   - **文件夹**：将文档归属到某个文件夹（可选）
   - **标签**：为文档添加标签（可选）
3. 拖拽或选择文件上传
4. 点击「开始上传」

### 3.3 文档状态说明

文档上传后会进入处理流程，状态含义如下：

| 状态 | 说明 |
|------|------|
| ⏳ **处理中** | 文档正在解析和切片 |
| 🔄 **向量生成中** | 正在生成文本向量 |
| 🕸 **图谱构建中** | 正在抽取实体和关系 |
| ✅ **就绪** | 文档处理完成，可被检索和问答使用 |
| ❌ **失败** | 处理失败，可查看错误日志 |

**文档处理流程：**

```
上传 → MinIO 存储
    → BullMQ 队列
    → 1. 解析 (PDF/Office/Markdown/TXT/Image OCR/Audio ASR)
    → 2. 切片 (500 tokens, 50 overlap)
    → 3. 向量化 (text-embedding-v4)
    → 4. 实体抽取 (LLM)
    → 5. Neo4j 图谱创建
    → 就绪
```

### 3.4 管理文档

**列表功能：**
- **搜索**：按文件名搜索
- **筛选**：
  - 按状态筛选（处理中 / 就绪 / 失败）
  - 按文件夹筛选
  - 按标签筛选
- **排序**：按上传时间、名称、状态排序

**操作：**
- **查看**：点击文件名查看文档详情
- **删除**：点击删除图标移除文档（同时删除向量和图谱数据）
- **预览**：PDF 文件可直接在线预览

### 3.5 文件夹管理

在左侧边栏「文件夹」区域：
- 新建文件夹
- 重命名文件夹
- 删除文件夹（需先移除内部文档）
- 将文档拖拽到文件夹中

### 3.6 标签管理

在左侧边栏「标签」区域：
- 创建新标签（可选择颜色）
- 为标签重命名
- 删除标签
- 点击标签筛选对应文档

## 4. 智能检索

### 4.1 页面入口

点击左侧导航栏「智能检索」进入检索页面。

### 4.2 检索模式

系统提供三种检索模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **关键词检索** | 基于 PostgreSQL 全文搜索 (BM25 + trigram) | 精确术语、专业词汇 |
| **语义检索** | 基于 pgvector 向量相似度 | 语义理解、模糊查询 |
| **混合检索** | BM25 + 向量 + RRF 融合 | 最佳平衡，推荐优先使用 |

### 4.3 检索步骤

1. 在搜索框输入查询关键词
2. 选择检索模式（默认「混合检索」）
3. 选择排序方式：按相关性、时间或名称
4. 选择返回数量（Top 5/10/20/50）
5. 点击搜索或按回车
6. 查看结果列表

### 4.4 结果展示

检索结果包含：
- **文档标题**
- **匹配片段**（安全高亮关键词）
- **匹配分数**（相关性评分）
- **标签和文件夹信息**
- **时间信息**
- **命中来源**（BM25 / 向量 / trigram）

### 4.5 检索参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| BM25 TopK | 关键词召回数量 | 50 |
| 向量 TopK | 语义召回数量 | 50 |
| RRF K 值 | RRF 融合参数 | 60 |
| 最终 TopK | 返回结果数量 | 10 |
| sortBy | 结果排序：相关性 / 时间 / 名称 | relevance |

### 4.6 检索历史

- 搜索后会按当前租户和用户记录检索历史
- 历史项会保存关键词、检索模式、排序方式、TopK、结果数量和时间
- 点击历史项可用原参数快速重搜
- 可删除单条历史，也可清空当前用户历史

## 5. AI 问答

### 5.1 页面入口

点击左侧导航栏「AI 问答」进入问答页面。

### 5.2 提问方式

1. 在输入框输入问题
2. 点击发送按钮或按回车
3. 系统会显示「正在思考...」状态
4. AI 以流式方式返回回答

### 5.3 会话管理

**新建对话：**
- 点击左上角「新建对话」按钮
- 输入对话标题（可选）

**切换对话：**
- 在左侧会话列表中选择历史对话

**继续追问：**
- 在当前会话中继续输入问题，系统会带上会话 ID
- 后端会结合最近对话历史改写检索问题，用于处理「这个」「继续展开」等追问

**删除对话：**
- 悬停在对话上，点击删除图标

### 5.4 引用来源

AI 回答时会自动标注引用来源：
- 回答中的 `[1]` `[2]` 等标记对应参考文档
- 回答下方显示引用卡片，包含：
  - 文档名称
  - 匹配内容片段
  - 相似度分数
- 点击引用卡片可跳转到原始文档

### 5.5 满意度反馈

- 每条助手回答下方提供点赞和点踩按钮
- 点赞/点踩后可填写可选文字反馈
- 再次点击同一按钮可撤销反馈
- 反馈按会话 owner 和租户隔离，仅能评价助手回答

### 5.6 无结果改写推荐

- 当问答检索不到相关资料时，页面会展示推荐改写问题
- 点击推荐问题会在当前会话中继续提问
- 该机制不会中断原回答，AI 仍会说明知识库资料不足并给出可用建议

### 5.7 技术原理

```
用户问题
    ↓
结合会话历史改写追问
    ↓
混合检索 (topK=5)
    ↓
无结果时生成改写建议
    ↓
组装 Prompt（含 [1][2] 引用标记）
    ↓
调用 qwen-plus 大模型
    ↓
流式返回 (SSE)
    ↓
前端渲染 + 显示引用卡片
```

## 6. 知识图谱

> 2026-07-06 更新：知识图谱工作台已增强为可解释、可治理、可追溯的 2D 图谱。Neo4j 负责图查询和可视化；PostgreSQL 中的 `knowledge_nodes`、`knowledge_edges`、`edge_evidences` 和图谱审计/视图表作为证据、治理与审计权威源。

### 6.0 当前图谱工作台能力

- 筛选与定位：支持关键词、节点类型、业务分类、文档、实体类型、关系类型、创建时间、更新时间、关系深度和节点数上限；搜索命中节点会在画布中高亮。
- 证据查看：点击节点或关系打开右侧证据面板，查看来源文档、chunk、证据原文、置信度、来源数量、关系权重和审核状态。
- 路径查询：在「路径与人工关系」面板选择起点、终点和最大跳数，查询两点最短路径并高亮路径子图。
- 保存视图：保存当前筛选条件和中心节点布局，支持仅自己可见或团队共享；点击已保存视图可恢复，垃圾桶按钮可删除。
- 导出：顶部导出按钮支持 PNG、SVG、JSON；JSON 包含图数据、筛选条件、中心节点和导出元数据。
- 实体治理：editor/admin 可维护实体别名，将重复实体软合并到目标实体；图谱默认隐藏 merged 源实体。
- 关系治理：editor/admin 可新增人工关系、编辑关系类型、删除关系，并将关系审核为 `APPROVED` / `PENDING` / `REJECTED`。
- 审计与同步：治理写操作按租户隔离，记录图谱变更审计，并同步 PostgreSQL 与 Neo4j。

### 6.1 页面入口

点击左侧导航栏「知识图谱」进入图谱可视化页面。

### 6.2 图谱展示

页面采用 **力导向图** 布局，展示：
- **圆形节点**：文档
- **方形节点**：实体（人名、地点、组织、概念等）
- **连线**：实体之间的关系

### 6.3 交互操作

| 操作 | 说明 |
|------|------|
| **拖拽** | 拖动节点调整布局 |
| **缩放** | 滚轮缩放视图 |
| **平移** | 拖拽空白区域平移 |
| **悬停** | 悬停节点显示详情 |
| **点击** | 点击节点查看详情面板 |
| **双击** | 双击实体节点展开关联节点 |

### 6.4 筛选功能

- **按关键词搜索**：高亮匹配的节点
- **按实体类型筛选**：只显示特定类型的实体
- **按深度筛选**：控制展开层级（1-3 层）

### 6.5 节点详情

点击节点显示详情面板：
- **文档节点**：文件名、状态、标签、上传时间
- **实体节点**：实体名称、类型、出现次数、关联文档列表

## 常见问题

### Q: 文档上传后一直显示「处理中」怎么办？

**可能原因：**
1. Redis / BullMQ 未正常运行
2. Worker 未启动
3. 文档过大或格式异常

**解决方法：**
1. 检查 Docker 容器状态：`docker compose ps`
2. 查看后端日志：`docker compose logs api`
3. 尝试删除并重新上传文档

### Q: 检索没有返回结果？

**可能原因：**
1. 文档状态不是「就绪」
2. 索引未完成

**解决方法：**
1. 检查文档状态是否全部变为「就绪」
2. 等待处理完成后再检索
3. 在问答页点击推荐改写问题，或换用更具体/更宽泛的关键词重搜

### Q: AI 问答引用不准确？

**可能原因：**
1. 相关文档未处理完成
2. 检索参数配置不当

**解决方法：**
1. 确保文档状态为「就绪」
2. 在设置中调整 topK 参数
3. 尝试使用「混合检索」模式

### Q: 图片或扫描 PDF 处理失败怎么办？

**可能原因：**
1. PaddleOCR 服务未启动或 `PADDLEOCR_HTTP_URL` 不可访问
2. 首次启动 PaddleOCR 正在下载模型
3. 扫描 PDF 页数较多，OCR 处理超时

**解决方法：**
1. 执行 `.\services\paddleocr-server\start.ps1` 并检查 `http://localhost:10096/health`
2. 中文文档保持 `PADDLEOCR_LANG=ch`
3. 调整 `PADDLEOCR_TIMEOUT_MS`、`OCR_PDF_RENDER_SCALE` 或 `OCR_PDF_MAX_PAGES`

### Q: 知识图谱没有数据？

**可能原因：**
1. 文档处理未完成「图谱构建」阶段
2. 实体抽取失败

**解决方法：**
1. 检查文档状态
2. 确认 LLM 服务可用
